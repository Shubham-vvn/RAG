Problem Statement: AI-Powered Rental Property Recommendation System
Problem Statement
You are tasked with building an AI-powered rental property recommendation service for the Indian housing market. The system should intelligently suggest rental listings based on user preferences by combining structured data with a Large Language Model (LLM).
Objective
Design and implement an application that:

Takes user preferences (such as city, budget, BHK configuration, furnishing status, and tenant type)
Uses a real-world dataset of rental listings
Leverages an LLM to generate personalized, human-like recommendations
Displays clear and useful results to the user

System Workflow
1. Data Ingestion

Load and preprocess the House Rent Prediction Dataset from the local path: /Users/shubhamthakur/Downloads/nextleap antigravity projects/04. House Rental Project/House_Rent_Dataset.csv
Extract relevant fields such as BHK, Rent, Size, Floor, Area Type, Area Locality, City, Furnishing Status, Tenant Preferred, Bathroom, and Point of Contact
Clean and normalize messy fields (e.g., parse "Floor" strings like "2 out of 5" into numeric floor and total-floor values; handle nulls and duplicate listings)
Since the raw dataset contains only structured numeric/categorical fields, generate a synthetic enrichment layer using an LLM — a short listing description, a simulated tenant review, and an inferred amenities list per row — to create realistic unstructured text for the reasoning layer to work on

2. User Input
Collect user preferences:

City (e.g., Mumbai, Bangalore, Delhi, Chennai)
Budget range (min–max monthly rent)
BHK configuration (1, 2, 3, 4+)
Furnishing status (Furnished, Semi-Furnished, Unfurnished)
Tenant type (Bachelors, Family, Company)
Any additional preferences (e.g., "quiet locality," "good for families," "close to IT hub," "spacious")

3. Integration Layer

Filter and prepare relevant rental listings based on user input using structured filtering (city, rent range, BHK, furnishing status, tenant preference)
Select a shortlist of top-matching candidates from the filtered results
Pass this shortlist — along with their synthetic descriptions, reviews, and amenities — into an LLM prompt
Design a prompt that helps the LLM reason over the soft preferences (the ones not captured by structured columns) and rank the shortlist accordingly

4. Recommendation Engine
Use the LLM to:

Rank shortlisted properties based on overall fit to both hard and soft preferences
Provide explanations (why each recommendation fits the user's stated needs)
Optionally summarize trade-offs across the top picks (e.g., "this option is cheaper but further from a furnished state")

5. Output Display
Present top recommendations in a user-friendly format:

Locality & City
BHK / Size / Furnishing Status
Monthly Rent
AI-generated explanation of fit

Dataset Notes (Important Constraints)

Source file: House_Rent_Dataset.csv (~4700+ listings), located at the local path specified above
Unlike datasets with native review or amenity text, this dataset is purely structured. This means:

The hard filtering step (city, rent, BHK, furnishing) is fully reliable and should be done with pandas/SQL logic, not the LLM
The soft reasoning step (locality vibe, tenant-fit, amenities) depends entirely on the synthetic enrichment layer you generate — this should be created once, offline, and cached to a new CSV, not regenerated on every user query
